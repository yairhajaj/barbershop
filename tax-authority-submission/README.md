# 🏛 הגשת תוכנה ל-רשות המיסים — חבילת מסמכים

תיקייה זו מכילה את כל המסמכים שעליך להגיש לרשות המיסים כדי לרשום את תוכנת
**Barbershop Booking** ולקבל **מספר רישום תוכנה** בן 8 ספרות.

## 📄 המסמכים בתיקייה

| # | קובץ | תפקיד |
|---|------|-------|
| 1 | `01-application.html` | **טופס בקשה לרישום תוכנה** — המסמך הראשי |
| 2 | `02-cover-letter.html` | **מכתב מלווה** לפתיחה רשמית של הבקשה |
| 3 | `03-compliance-declaration.html` | **הצהרת עמידה** בדרישות 24/2004 ו-1.31 |
| 4 | `04-software-use-notification.html` | **הודעה על שימוש בתוכנה** (§18ב(ב)) — מוגשת *אחרי* קבלת מספר הרישום |

## 🖨 איך להשתמש

1. פתח כל קובץ בדפדפן (דאבל-קליק).
2. לחץ על הכפתור הזהוב **"הדפס / שמור כ-PDF"** בפינה העליונה.
3. בחר **"שמור כ-PDF"** כיעד ההדפסה.
4. מלא את השדות הריקים בכתב יד או בעורך PDF.
5. חתום במקום הנדרש.

## 📝 מה עוד צריך להכין

לפני ההגשה, הכן גם:

- [ ] **דוגמת קובץ אחיד** — הפק מהאפליקציה (AccountantTab → OPENFRMT). אל תדאג אם אין לך מספר רישום עדיין — הקובץ יופק עם "0" ורק ישמש כדוגמה.
- [ ] **צילומי מסך מהאפליקציה** — 3-4 צילומים: מסך הגדרות רישום תוכנה, מסך הפקת קבצים, דוח סעיף 2.6, דוגמת חשבונית PDF.
- [ ] **קישור לאפליקציה** — URL של האתר + לינקים ל-App Store ו-Google Play (אם כבר פורסמו).
- [ ] **שאלון ביטחון/פרטיות** — אופציונלי, אם הרשות תבקש: תיאור של אבטחת מידע (TLS, RLS, גיבויים).

## 📮 לאן לשלוח

אחת משלוש אפשרויות:

### אפשרות א — דואר רשום
> רשות המיסים בישראל — אגף מע״מ
> המחלקה למערכות מידע חשבונאיות
> דרך בר-יהודה 5, ירושלים 9103701

### אפשרות ב — דוא״ל
> software-reg@taxes.gov.il
> (יש לוודא כתובת עדכנית בטלפון **4954*** — מרכז שירות לקוחות רשות המיסים)

### אפשרות ג — פגישה אישית
ניתן לתאם פגישה עם המחלקה להצגת האפליקציה ולבצע הגשה פיזית.
דרך זאת מומלצת לפעמים לתוכנות חדשות שטרם רשומות.

## ⏳ זמני תגובה צפויים

- **תשובה ראשונית:** 2-4 שבועות
- **אם יידרשו תיקונים:** הרשות תשלח מכתב עם דרישות; תיקון → הגשה מחדש.
- **אישור סופי + מספר רישום:** 4-8 שבועות מהגשה ראשונה.

## ✅ אחרי קבלת מספר הרישום

1. הכנס את המספר ב-**הגדרות → רישום תוכנה ברשות המיסים → מספר רישום תוכנה**.
2. סמן את **"הודעת שימוש בתוכנה נמסרה לרשות המיסים"**.
3. שלח את `04-software-use-notification.html` למשרד השומה האזורי שלך.
4. מרגע זה, כל קובץ אחיד שיופק יכלול את המספר הנכון בשדה 1006 של A000.

## 🧪 בדיקה לפני הגשה

הפק קובץ אחיד מהאפליקציה והעלה אותו לסימולטור הממשלתי:

🔗 **https://www.gov.il/he/service/download-open-format-files**

הסימולטור יאמת:
- אורכי רשומות (466/95/444/339/222/298/110)
- קיום קוד המערכת `&OF1.31&`
- תקינות Primary ID (15 ספרות)
- ספירת רשומות ב-Z900
- קידוד Windows-1255 + CRLF

אם הסימולטור עובר — סביר להניח שגם הרשות תאשר.

**תשובה:** לא חובה — אפשר להגיש ישירות כיצרן. עם זאת, אם אתה בעל חברה, מומלץ לתאם עם רואה החשבון כדי שיידע על הרישום.

**תשובה:** אפשר להגיש בקשה ולציין ״גרסה 1.0-beta״. הרשות תבחן לפי מה שמוגש.

**תשובה:** שינוי גרסה רגיל (bug fixes, features חדשות) — לא. שינוי מהותי במבנה הקובץ האחיד או בלוגיקת ההפקה — כן, יש להודיע.
